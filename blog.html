Important FAQ

What is Document Object Model (DOM)?

The Document Object Model (DOM) is a programming interface for web documents. It represents the structure of a webpage so that it can be accessed and manipulated by programming languages, such as JavaScript. The DOM treats the entire HTML or XML document as a tree of objects, where each object represents a part of the document (like an element, attribute, or text).

How do you select an element from the DOM?

To select an element from the DOM using JavaScript, there are several methods depending on how you want to target the element (by ID, class, tag, or selector). Here are the most commonly used DOM selection methods:

  • Use document.getElementById() to select an element by its unique ID.
  • Use document.getElementsByClassName() to select elements by their class name. This method returns an HTMLCollection (a collection of elements).
  • Use document.getElementsByTagName() to select elements by their tag name.This also returns an HTMLCollection.
  • Use document.querySelector() to select the first element that matches a specific CSS selector (e.g., ID, class, tag, or more complex CSS selectors).
  • Use document.querySelectorAll() to select all elements that match a specific CSS selector. This method returns a NodeList, which is similar to an array.

    • What is event delegation in the context of the DOM, and why is it useful?

    Event delegation is a technique in the DOM that allows you to handle events at a higher level in the DOM tree rather than attaching event listeners to each individual element. Instead of attaching an event listener to every child element, you attach a single event listener to a parent element, which listens for events triggered by its children due to the way events propagate in the DOM.This technique is particularly useful in scenarios where:

    1. 1. You have many child elements that need to handle the same event.
    2. 2. You expect the number of child elements to grow dynamically (e.g. when adding elements through JavaScript).

    How do you manipulate an elements style & attribute using the DOM?

    To change the appearance of an element, you can modify its CSS properties. This could include adjusting things like text color, background color, font size, or hiding the element entirely. Instead of editing a style sheet, you directly assign these styles to the element, typically for dynamic or interactive effects.

  • Manipulating Classes: CSS classes are pre-defined style rules that can be applied to elements. Instead of manually setting each style, you can add or remove CSS classes. This allows you to apply or revoke a whole set of styles at once.
  • Manipulating Attributes: HTML elements come with attributes like src (for images), href (for links), or alt (for image descriptions). You can modify these attributes to change the element's behavior or content.
  • Manipulating data-* Attributes: These are custom attributes that store extra information on elements, usually for use with JavaScript. You can dynamically assign values to these attributes or read their values to interact with the element based on the data.
